Core Concepts

🔒 Password Hashing

A hash function converts a password into a fixed-length string. It's one-way — you can't reverse it. When you login, the system hashes your input and compares it to the stored hash.

SHA256 · MD5 · SHA512

📖 Dictionary Attack

Attackers hash thousands of common passwords and compare to the target hash. If you use "password123", it will be cracked in milliseconds — it's in every wordlist.

Wordlist · Pre-computation

💪 Brute-Force Attack

Try every possible combination of characters. Slow but exhaustive. A 6-char lowercase password has 26⁶ = 308 million combinations. With special chars and length 12+, it's infeasible.

Combinatorial · Exhaustive

🧂 Salting

A random string added to the password before hashing. Even if two users have the same password, their hashes differ. Defeats rainbow tables and pre-computation attacks.

Rainbow Tables · Uniqueness

🌈 Rainbow Tables

Pre-computed tables mapping hash → password. Attackers can look up MD5("password") instantly. Salting destroys this — they'd need a separate table per salt.

Lookup · Time-Memory Tradeoff

🐌 Slow Hashing (bcrypt)

MD5/SHA256 are designed to be fast — bad for passwords. bcrypt, Argon2, scrypt are intentionally slow (100ms+), making brute-force billions of times harder.

bcrypt · Argon2 · scrypt · PBKDF2
